# Vendor-Performance-Analysis
📊 Vendor Performance Analysis

This project focuses on analyzing vendor performance using data analytics techniques to uncover key business insights and support data-driven decision-making.

The analysis involves collecting and integrating data from multiple sources such as purchases, sales, and vendor invoices, followed by data cleaning, transformation, and exploratory data analysis (EDA). Key performance indicators (KPIs) such as total purchases, total sales, profit margin, stock turnover, and sales-to-purchase ratio are calculated to evaluate vendor efficiency.

Advanced analysis techniques such as Pareto Analysis (80/20 rule) are used to identify top-performing vendors contributing the most to overall purchases. Additionally, statistical hypothesis testing (t-test) is performed to compare profit margins between high-performing and low-performing vendors.

The project also includes data visualization using Python libraries like Matplotlib and Seaborn to present insights through bar charts, Pareto charts, and comparative plots. The final processed data is stored in a database and can be further used for dashboard creation in tools like Power BI.

🚀 Key Features

Data cleaning and preprocessing

SQL-based data extraction and transformation

KPI calculation (profit margin, turnover, ratios)

Pareto analysis for vendor contribution

Hypothesis testing (t-test)

Data visualization (Matplotlib & Seaborn)

Database integration

🛠️ Tech Stack

Python (Pandas, NumPy)

SQL (PostgreSQL / SQLite / MySQL)

Matplotlib & Seaborn

Power BI (for dashboarding)

📈 Outcome

The project helps identify:

Top-performing vendors

Key contributors to overall business performance
